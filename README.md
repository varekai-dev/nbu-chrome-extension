# NBU Coins - Auto Add

Chrome extension для автоматичного моніторингу та додавання товарів з каталогу Національного банку України в кошик.

## Особливості

- **Два режими роботи** — каталог/головна сторінка (пошук за назвою) та сторінка окремого товару (пряме натискання кнопки "Купити")
- **Автоматичний моніторинг** — клікає на кнопку як тільки вона з'являється в DOM
- **Auto-refresh** — оновлює сторінку з налаштованим інтервалом (210–1210 мс)
- **Запланований старт** — можна задати точний час запуску до секунди
- **Збереження налаштувань** — назва товару, інтервал та запланований час зберігаються між сесіями

## Встановлення

1. Завантажте всі файли проекту в одну папку
2. Відкрийте `chrome://extensions/`
3. Увімкніть **Developer mode** (правий верхній кут)
4. Натисніть **Load unpacked** і виберіть папку з файлами
5. Опціонально: закріпіть іконку в toolbar через меню 🧩

## Використання

### Режим каталогу (головна або `/catalog.html`)

1. Перейдіть на [coins.bank.gov.ua](https://coins.bank.gov.ua/catalog.html)
2. Відкрийте popup extension
3. Введіть **точну назву товару** (наприклад, "Архістратиг Михаїл")
4. За потреби змініть **інтервал оновлення** (за замовчуванням 1210 мс)
5. За потреби вкажіть **запланований час** старту — або залиште порожнім для негайного старту
6. Увімкніть **toggle**

### Режим сторінки товару (`/p-NNN.html` та інші)

1. Відкрийте сторінку конкретного товару
2. Відкрийте popup extension — поле назви товару сховане (ви вже на потрібній сторінці)
3. За потреби змініть **інтервал оновлення** або вкажіть **запланований час**
4. Увімкніть **toggle** — extension чекатиме появи кнопки «Купити» і клікне її

### Загальна поведінка toggle

- Без запланованого часу → моніторинг стартує одразу
- З запланованим часом → стартує автоматично у вказаний момент, навіть якщо popup закритий
- Вимкнення toggle скасовує і активний моніторинг, і запланований час

## Як це працює

При завантаженні сторінки content script визначає режим за URL:

**Каталог / головна** (`/`, `/catalog.html`):
1. Шукає товар за точною назвою серед елементів `.product`
2. Якщо знайдено і кнопка активна (не `.gray`, не `.clicked`) — клікає і зупиняє моніторинг
3. Якщо не знайдено або кнопка недоступна — чекає наступного перезавантаження

**Сторінка товару** (будь-який інший шлях на домені):
1. Шукає `button.btn-primary.buy` в DOM
2. Якщо кнопка є — одразу клікає і зупиняє моніторинг
3. Якщо кнопки немає (товар недоступний) — чекає наступного перезавантаження

В обох режимах:
- Сторінка перезавантажується з заданим інтервалом доки товар не куплено або toggle не вимкнено
- Запланований час зберігається в `chrome.storage.local`; content script встановлює таймер і автоматично стартує без відкритого popup


## Chrome Storage

| Ключ | Тип | Опис |
|---|---|---|
| `filterText` | string | Назва товару для пошуку |
| `toggleEnabled` | boolean | Стан моніторингу |
| `refreshIntervalMs` | number | Інтервал оновлення (210–1210) |
| `scheduledTime` | string | Запланований час старту (datetime-local format) |

## Важливо

- Extension працює тільки на `coins.bank.gov.ua`
- В режимі каталогу пошук за **точною** назвою товару (з нормалізацією пробілів)
- Сторінка постійно оновлюється поки toggle увімкнений — це нормальна поведінка
- Кошик зберігається між перезавантаженнями

## Усунення проблем

**Extension не реагує** — перевірте що ви на `coins.bank.gov.ua` і extension увімкнено в `chrome://extensions/`

**Товар не знаходиться (каталог)** — переконайтесь що назва введена точно так само як на сайті (регістр важливий)

**Toggle не вмикається (каталог)** — поле назви товару не може бути порожнім

**Extension не клікає на сторінці товару** — переконайтесь що сторінка не є `/catalog.html` і не є головною; перевірте що toggle увімкнено

**Після змін у коді** — відкрийте `chrome://extensions/`, натисніть 🔄 на розширенні, перезавантажте вкладку
