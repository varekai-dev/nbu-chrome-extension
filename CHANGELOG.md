# Changelog

## [1.1.0] - 2026-04-13

### Додано
- Підтримка сторінок окремих товарів (`/p-NNN.html`) — extension тепер автоматично клікає `<button class="btn-primary buy">Купити</button>` як тільки вона з'являється в DOM
- URL-детекція режиму: каталог/головна (`/`, `/catalog.html`) — стара логіка; будь-яка інша сторінка домену — режим товару
- На сторінці товару поле "Назва товару" автоматично ховається в popup (toggle і розклад залишаються)

### Змінено
- Toggle на сторінці товару вмикається без введення назви товару

---

## [1.0.9] - 2026-04-13

### Додано
- Datepicker (`datetime-local` з точністю до секунди) для запланованого старту моніторингу
- Числовий інпут для налаштування інтервалу оновлення сторінки (210–1210 мс, за замовчуванням 1210)
- Кнопка × для очищення запланованого часу
- Стан "Заплановано о HH:MM:SS" в індикаторі toggle

### Змінено
- Toggle тепер враховує запланований час: якщо час введено — старт по розкладу, якщо ні — одразу
- Вимкнення toggle скасовує і активний моніторинг, і запланований час
- Інтервал оновлення сторінки тепер налаштовується через UI замість захардкодженого значення
- При відкритті popup запланований час що вже минув автоматично очищується

### Видалено
- 500 мс інтервал повторної перевірки DOM (`setInterval checkProduct`) — контент статичний між перезавантаженнями, перевірка при кожному завантаженні достатня
